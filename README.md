# Project Forge

## プロジェクト概要

Project Forgeは、統計的優位性に基づく金融取引システムの構築を目指す研究開発プロジェクトです。
最終目標である「Project Chimera（金融知性体）」の実現に向けた資金獲得を主目的としています。

本プロジェクトの思想的支柱は、ルネサンス・テクノロジーズの哲学に根ざしており
人間の先入観を排除し、市場に潜む統計的に有意なパターン（「マーケットの亡霊」）のみを追求します。

## 対象市場

- **主戦場**: XAU/USD
- **将来展開**: USD/JPY等の相関データ活用を視野

## 基本哲学

### 設計思想の進化

当初計画していた手動非線形変換（乗算や多項式）による特徴量空間の拡張から、重要な発見により方針を転換しました。

**核心的理解**: LightGBM自体が最強の非線形処理エンジンである

LightGBMは内部で数千の決定木を構築し、複雑な条件分岐を自動的に最適化します。これにより：

1. 手動での非線形変換は不要
2. アーキテクチャの大幅な簡素化を実現
3. ストレージ要件を劇的に削減（TB級 → GB級）

新しいアーキテクチャは「統計的品質管理による段階的選別（Sequential Quality-Based Selection）」を基本哲学としています。

## データ基盤

### 入力データ仕様

- **総行数**: 約1億5000万行のTickデータ
- **時間軸**: 15種類の時間足（tick, M0.5, M1, M3, M5, M8, M15, M30, H1, H4, H6, H12, D1, W1, MN）
- **データサイズ**: 300GB〜500GB
- **保存形式**: Hiveパーティション化されたParquet形式
- **データソース**: `/workspaces/project_forge/data/1_XAUUSD_base_data/XAUUSD_tick_master_GOLDEN`

### ベース特徴量（生成済み）

総数324個の特徴量を体系的に生成済み：

**1A. 基礎統計・ロバスト統計（約60個）**
- 統計的モーメント、ロバスト統計、統計検定

**1B. 時系列分析・分布パラメータ（約60個）**
- 単位根検定、構造変化検定、分布フィッティング、予測モデル

**1C. テクニカル指標・トレンド分析（約70個）**
- RSI、MACD、ボリンジャーバンド、各種移動平均線等

**1D. 出来高・価格アクション・サポレジ（約70個）**
- ボラティリティ指標、出来高指標、ブレイクアウト系、ローソク足パターン

**1E. 信号処理・周波数解析（約30個）**
- スペクトル特徴、ウェーブレット、ヒルベルト変換

**1F. 学際的・実験的特徴量（約16個）**
- ネットワーク科学、言語学、美学、音楽理論、生体力学

**高度特徴量**
- MFDFA（マルチフラクタル除トレンド変動解析）: 9個
- Kolmogorov複雑性: 9個

**将来の拡張候補**: 51の高度特徴量（カオス理論系、位相幾何学系等）

## コアアーキテクチャ

### カーブフィッティング三重防衛網

膨大なベース特徴量から、統計的に信頼できる真のアルファを持つエリート特徴量のみを選抜する多段階検証システム。

#### 第一防衛線：時間的安定性の確保

**スクリプト**: `feature_validator.py`

**目的**: 時間経過で性質が変化する不安定な特徴量を早期排除

**検査内容**:
1. 分布安定性テスト（KS検定）- 前半/後半期間での分布比較
2. 重要度安定性テスト（PI安定性）- パーミュテーション重要度の期間間変動分析

**生存率**: 全特徴量 → 数百個（推定）

#### 第二防衛線：AI監督による統合的チーム編成

**スクリプト**: `walk_forward_validator.py`

**目的**: LightGBMによる実戦で最も活躍できる最強チームの選抜

**LightGBMの統合的判断**:
- チームプレイ: 複数特徴量の複雑な条件分岐
- スーパースター単独判断: 単一特徴量による直滑降
- 確信度ブースター: 強力な特徴量による確信度向上

**実装**: TimeSeriesSplitによるウォークフォワード・クロスバリデーション

**生存率**: 数百個 → 数個〜数十個（推定）

#### 第三防衛線：アルファの純化

**スクリプト**: `feature_neutralizer.py`

**目的**: 市場ベータの除去による真のアルファの抽出

**実装**: 線形回帰による市場ベータ除去、残差としてのアルファ抽出

#### 統合マスターテーブル構築

**スクリプト**: `build_master_table.py`

第一防衛線通過特徴量を統合した最終選考用テーブルを作成。join_asofによる時間軸統合。

## 予測AIコア

### トリプルバリア・メソッド

実際の取引における損益発生メカニズムを反映：

- 利食いバリア: 現在価格 + 2 × ATR
- 損切りバリア: 現在価格 - 1 × ATR
- 時間バリア: 最大保有期間（例: 60バー）

### デュアルモデル・アーキテクチャ

**モデル1（LGBMClassifier）**: バリア到達結果の予測（確信度）

**モデル2（LGBMRegressor）**: バリア到達時間の予測

### ハイパーパラメータ最適化

Optunaによる効率的探索とTimeSeriesSplitによる時系列妥当性検証の統合。

## 執行システム

### 極限リスク管理エンジン

**スクリプト**: `extreme_risk_engine.py`

- Exnessゼロカットシステムをセーフティネットとして活用
- AIシグナル確率に基づく動的ポジションサイズ調整
- GARCHボラティリティ予測統合
- 最大ドローダウン制約実装

### MQL5ブリッジ：ZeroMQアーキテクチャ

PythonベースAIコアとMQL5取引ターミナル間の非同期通信。

- Python発行者: JSON形式取引コマンド発行
- MQL5購読者: シグナル購読と注文実行

疎結合アーキテクチャによる保守性とスケーラビリティの確保。

## 技術スタック

### 開発環境

- **言語**: Python
- **主要ライブラリ**: Polars（データ処理）、NumPy、Numba（JIT最適化）
- **開発ツール**: Visual Studio Code
- **実行環境**: Dev Container（Docker）

### ハードウェア要件

- NVIDIA GeForce RTX 3060 12GB
- Intel i7-8700K 6コア12スレッド
- NVMe SSD 1TB
- 64GB RAM

### アーキテクチャ原則

**3クラス構成**:

1. **DataEngine（30%）**: Parquetメタデータ検証、遅延読み込み、述語プッシュダウン最適化
2. **CalculationEngine（60%）**: Polars Expressionによる標準計算、Numba UDFによる専門計算
3. **OutputEngine（10%）**: ストリーミング出力、メタデータ記録

**データ処理戦略**:

- 標準アーキテクチャ: エンドツーエンド処理（M1, H1, D1等）
- 大規模データ用: 分割・重複・処理・結合（Tickデータ専用）

### 設計原則

**革新性と確実性の分離**:
- 革新性追求領域: 特徴量計算アルゴリズムのみ（学術論文レベル）
- 確実性重視領域: システム基盤全て（枯れた技術のみ）

**計算アルゴリズムの厳密性**:
- 近似式の禁止
- 厳密な定義の遵守
- UDFの堅牢性確保（ゼロ除算対策等）

## ストレージ要件

手動非線形処理廃止により大幅削減：

- ベース特徴量（永続）: 400 GB
- 安定版マスターテーブル（永続）: 50〜80 GB
- エリート特徴量（永続）: 5〜15 GB
- 一時領域: ほぼ不要
- OSその他: 100 GB

**推奨**: 1TB NVMe SSD（最低600GB空き容量）

## プロジェクト背景

本プロジェクトは、個人投資家が市場で持続的に成功するための唯一の合理的アプローチとして構想されました。
人間の認知バイアスと限界を認識し、客観的かつ体系的な定量的アプローチによって、再現性のない「裁量の天才」の領域を、再現可能なサイエンスへと昇華させることを目指しています。

開発期間わずか2ヶ月（Python学習開始から1.5ヶ月）で、個人投資家の上位0.01%〜0.001%の領域に到達する技術基盤を構築しました。
